Connect to shared database

Make this token a fine-grained PAT, not a classic token (GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens):

  • Grant Contents: Read and write only — no other permissions.
  • Scope it to the one shared repo ("Only select repositories"), never all repos.
  • Set an explicit expiry so a leaked token self-limits.
  • The shared repo holds only the 3 CSVs + the .mise-lock.json lock file — nothing else.
  • Never commit this shared token — or any Anthropic API key — into the shared repo.

Your name is prepended to every save's commit message so others can see who changed what. You can't save to the shared database until it's set.

This shared token is saved in this browser's localStorage as plain text. Only run this tool in a browser profile you control. A PUBLIC repo will be refused — it would expose your entire recipe database to anyone.

To rotate a leaked or expired token: Clear token (keep repo) → paste the new token above → Save & connect.
